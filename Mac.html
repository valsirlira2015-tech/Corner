📋 Registro de Endereços I2C

📌 Introdução ao I2C

O I2C (Inter-Integrated Circuit) é um barramento serial síncrono de curta distância que utiliza apenas duas linhas bidirecionais: SDA (Serial Data) e SCL (Serial Clock). [[76]]

O espaço de endereçamento de 7 bits permite teoricamente 128 endereços I2C, porém alguns endereços são reservados para propósitos especiais. Assim, apenas 112 endereços estão disponíveis para dispositivos. [[76]]

Dispositivo Endereço 7-bit Endereço 8-bit (R/W) Tipo
CD3215/CD3217 (USB-C PD) 0x42 / 0x43 0x84-0x87 OTP/Strap
SPD EEPROM (DDR4) 0x50-0x57 0xA0-0xAE Memória
SPD Hub (DDR5) 0x50 + HSA 0xA0-0xAF Memória
SMC (System Management) 0x2E 0x5C-0x5D Controlador
Sensor Térmico TMP100 0x48-0x4F 0x90-0x9F Sensor
HID over I2C 0x15-0x1F 0x2A-0x3F Input
Thunderbolt NHI 0x10-0x13 0x20-0x27 Controlador
General Call 0x00 0x00 Reservado
Nota: Os endereços I2C do CD3217 são configurados via OTP (One-Time Programmable) ou straps de depuração. Cada controlador recebe seu endereço único baseado na configuração de straps. [[1]]
🎯 Endereços Reservados SMBus
  • 0000 000x - General Call Address
  • 0000 001x - CBUS Address (Reservado)
  • 0000 010x - Reservado para uso futuro
  • 0000 011x - Reservado para uso futuro
  • 0001 100x - SMBus Alert Response Address
  • 0110 111x - ACCESS.bus Default Address
  • 1100 001x - SMBus Device Default Address
  • 1111 111x - High Speed Master Code

🔷 SMBus Principal

📖 O que é SMBus?

O System Management Bus (SMBus) é um barramento de controle simples de 2 fios usado para comunicação de baixa velocidade em sistemas eletrônicos. É baseado no protocolo I2C e é amplamente utilizado em computadores para comunicação com componentes de gerenciamento de energia e sensores. [[24]]

Importante: O SMBus possui requisitos de timeout que o I2C padrão não tem. Dispositivos SMBus devem liberar o barramento dentro de 25-35ms para evitar bloqueio do sistema. [[24]]
🔧 Protocolo SMBus

Formato de Pacote SMBus

Write Byte:
  [START] [Slave Addr + W] [ACK] [Command] [ACK] [Data] [ACK] [PEC] [ACK] [STOP]

Read Byte:
  [START] [Slave Addr + W] [ACK] [Command] [ACK] 
  [RESTART] [Slave Addr + R] [ACK] [Data] [ACK] [PEC] [ACK] [STOP]

Block Write:
  [START] [Slave Addr + W] [ACK] [Command] [ACK] [Byte Count] [ACK] 
  [Data1] [ACK] [Data2] [ACK] ... [DataN] [ACK] [PEC] [ACK] [STOP]
Dispositivo SMBus Típico Endereço Função
Battery Charger 0x09 / 0x12 Carregamento de bateria
Battery Fuel Gauge 0x0B Medição de carga da bateria
SPD EEPROM 0x50-0x57 Dados da memória RAM
Temperature Sensor 0x4C / 0x4D Monitoramento térmico
Voltage Regulator 0x58-0x5F Controle de tensão

💻 I2C SMC Interno

️ System Management Controller

O System Management Controller (SMC) é um subsistema encontrado em Macs Intel, responsável por gerenciar funções de baixo nível relacionadas a energia, térmica e firmware. [[88]]

O SMC controla todos os aspectos do fluxo de energia no sistema, incluindo resposta ao botão de energia, gerenciamento térmico, sensores, LEDs e muito mais. [[96]]

Nota: Macs com processadores Apple Silicon não possuem SMC tradicional. As funções são integradas no SoC principal. [[95]]
⚙️ Funções do SMC
  • Gerenciamento de energia e sequência de power-on
  • Controle de ventiladores e temperatura
  • Gerenciamento de bateria e carregamento
  • Controle de LEDs de status
  • Detecção de abertura/fechamento da tampa
  • Controle de retroiluminação do teclado
  • Gerenciamento de sensores de movimento (Sudden Motion Sensor)
  • Comunicação via I2C/SMBus com periféricos
🔍 Endereços I2C do SMC
Função Endereço I2C Descrição
SMC Primary 0x2E Controlador principal
SMC Secondary 0x2F Controlador secundário
Power Management 0x50-0x5F Dispositivos de energia

🔌 I2C UPC (USB-C)

🔋 CD3215/CD3217 - USB-C PD Controllers

O CD3215 é uma versão customizada do TPS65982/86 da Texas Instruments, configurado via OTP para uso em MacBooks e iPads. [[15]]

O CD3217 é a segunda geração (ACE2) de controladores USB-C PD da Apple, substituindo o CD3215 (ACE1). [[32]]

🚨 Problemas Comuns

CAUSACD3217 master (DFU port) defeituoso, ou problema de energia VIN_3V3 para os ACE2, inicializando o barramento I2C_UPC. [[55]]

VERIFICARVerificar PP3V3_UPC_LDO em todas as portas. Medir I2C_UPC com analisador lógico.

🔧 Diagnóstico por Sintoma

Uma porta específica não carrega, demais funcionam

CAUSACD3217 daquela porta está morto ou com firmware corrompido.

VERIFICARTrocar o CD3217 da porta afetada (vanilla se lado esquerdo, verificar OTP se lado direito). [[54]]

Carrega na bateria e no carregador separados, mas para quando ambos conectados

CAUSACD3217 com inrush limiter danificado (curto em OCP anterior ao Buck-Boost).

VERIFICARTrocar o CD3217 do conector USB-C que apresenta o comportamento — o inrush integrado é danificado por surtos de corrente.

Nenhuma porta funciona após substituição de CD3217

CAUSAConflito de endereço I2C — chip OTP instalado em posição errada, ou dois chips com mesmo endereço no barramento.

VERIFICARVerificar com I2C scanner se dois chips respondem no mesmo endereço. Realizar se o chip instalado é vanilla ou OTP e se o endereço OTP corresponde ao exigido pelo esquema.

Dica de Reparo: Ao substituir CD3217, sempre use chips da mesma revisão (B12, B13, etc.) e verifique se o EEPROM/ROM está programado corretamente. Chips "vanilla" precisam de programação OTP. [[56]]

🖱️ I2C HID

📱 HID over I2C Specification

HID over I2C fornece suporte para vários dispositivos de interface humana (teclados, touchpads, touchscreens) através do barramento I2C. A especificação foi desenvolvida pela Microsoft e é amplamente utilizada em dispositivos Windows e modernos. [[59]]

⚡ Características
  • Suporta I2C Fast Mode+ (até 1 MHz)
  • Protocolo baseado em descritores HID padrão USB
  • Redução de pinos comparado ao USB
  • Baixo consumo de energia
  • Plug-and-play via Device Tree/ACPI

💾 Memória I2C / SPD

💡 O que é SPD?

SPD (Serial Presence Detect) é uma EEPROM que armazena informações sobre o módulo de memória RAM, incluindo tipo (DDR3, DDR4, DDR5), velocidade, timings, capacidade e fabricante. A comunicação é feita via SMBus/I2C. [[70]]

📍 Endereçamento SPD

O endereço I2C completo é formado por: 1010(A2)(A1)(A0)b = 0x50-0x57

Os pinos A2, A1, A0 determinam qual slot de memória está sendo acessado. Para DDR4 SODIMM, o endereço base é 0x50. [[68]]

DDR5 utiliza um SPD Hub com endereço base 0x50, onde os bits 3-1 são configurados via pinos HSA (Hub Slave Address). [[72]]

Offset Descrição (DDR4) Valor Típico
0x00 SPD Revision 0x0C (DDR4)
0x02 DRAM Device Type 0x08 (DDR4)
0x04 Module Type 0x02 (SODIMM)
0x05 SDRAM Density & Bus Width Varia
0x10-0x13 Base DRAM Speed (MT/s) 0x19 (2400), 0x1D (2666)
0x14-0x17 Manufacturer 0xCE (Samsung), 0x2C (Micron)
0x7F SPD Checksum Checksum CRC
Atenção: A EEPROM SPD é crítica para o boot do sistema. Se corrompida, a memória não será detectada corretamente. Utilize programadores EEPROM para regravar dados SPD válidos. [[70]]

Cada interface I2C consiste em duas linhas seriais: SDA (Serial Data) e SCL (Serial Clock). Os controladores devem ser programados apenas para operar em modo iniciador (master). Modo target (slave) não é suportado. [[83]]

⚡ I2C Thunderbolt

🔌 Thunderbolt e I2C

O Thunderbolt 3/4 utiliza USB-C como conector físico e integra controladores USB-C PD (como CD3215/CD3217) para gerenciamento de energia e dados. A comunicação I2C é usada para configurar e monitorar os controladores Thunderbolt. [[34]]

Componente Endereço I2C Função
Thunderbolt NHI 0x10-0x13 Native Host Interface
Thunderbolt CIO 0x14-0x17 Controller I/O
USB-C PD (CD3217) 0x42-0x43 Power Delivery
Retimer 0x20-0x27 Signal conditioning
🔍 Diagnóstico Thunderbolt
  • Verificar alimentação 3.3V nos controladores
  • Medir linhas I2C_SDA e I2C_SCL (deve haver pull-up para 1.8V ou 3.3V)
  • Verificar comunicação com analisador lógico
  • Testar CC1/CC2 para detecção de cabo
  • Verificar firmware do controlador Thunderbolt

📌 Notas (Registro)

📝 Informações Importantes
  • Endereços I2C: Sempre use endereçamento de 7 bits em documentação. O bit R/W é adicionado automaticamente pelo hardware.
  • Pull-up Resistors: Valores típicos: 4.7kΩ para 3.3V, 2.2kΩ para 1.8V.
  • Capacitância: Máximo de 400pF por linha I2C para garantir integridade do sinal.
  • Velocidade: Standard Mode (100kHz), Fast Mode (400kHz), Fast Mode+ (1MHz), High Speed (3.4MHz).
  • Timeout: SMBus requer timeout de 25-35ms. I2C padrão não tem timeout definido.
Cuidado: Nunca conecte dispositivos com tensões diferentes diretamente no barramento I2C sem level shifters apropriados.

🔬 Análise Profunda ACE (CD3215/CD3217)

Visão Geral ACE

ACE (Apple Charging Engine) é a denominação dos controladores USB-C PD customizados da Apple. Existem duas gerações principais:

  • ACE1: CD3215 (primeira geração)
  • ACE2: CD3217B12, CD3217B13, CD3218B12 (segunda geração)
🔍 Arquitetura Interna

O CD3217 integra múltiplas funções:

  • Controlador USB Type-C PD (Power Delivery)
  • Gerenciamento de carregamento de bateria
  • Proteção contra sobrecorrente (OCP)
  • Proteção contra sobretensão (OVP)
  • Comunicação I2C/SMBus
  • Interface SPI para EEPROM/ROM
  • Reguladores LDO integrados
  • Buck-Boost converter control
⚡ Sequência de Power-On
  1. Aplicação de VIN_3V3 (3.3V) no CD3217
  2. Inicialização do firmware interno
  3. Leitura da configuração OTP/EEPROM
  4. Configuração das linhas I2C_UPC
  5. Habilitação dos LDOs de saída
  6. Monitoramento das linhas CC1/CC2
  7. Negociação USB PD com carregador
  8. Habilitação do VBUS para o sistema
Problema Comum: Se o CD3217 não tiver saídas LDO, teste as linhas para curtos. Caso contrário, o chip está sem alimentação ou defeituoso. [[55]]

⚖️ ACE1 vs ACE2

🔄 Diferenças Principais

ACE1 (CD3215)

  • Primeira geração de controladores USB-C Apple
  • Baseado no TI TPS65982/86
  • Configuração via straps de depuração
  • Cada controlador recebe endereço via straps
  • Usado em MacBook Pro 2016-2017 (A1706, A1707, A1708)
  • Suscetível a falhas de carregamento

ACE2 (CD3217)

  • Segunda geração com melhorias significativas
  • Suporte a USB PD 3.0 e PPS (Programmable Power Supply)
  • Configuração OTP (One-Time Programmable)
  • Endereços I2C mistos OTP/Strap
  • Usado em MacBook Pro 2018+ e iPads Pro
  • Maior confiabilidade e eficiência
  • Integração com chip T2 (em alguns modelos)
Compatibilidade: CD3215 e CD3217 NÃO são intercambiáveis devido a diferenças de pinout, firmware e configuração OTP. Sempre use o chip correto para o modelo. [[15]]

🛠️ Guia de Reparo CD3215

🔧 Procedimento de Substituição

1. Diagnóstico Inicial

  • Verificar tensão VIN_3V3 (deve ser 3.3V)
  • Medir linhas CC1/CC2 (deve haver pull-down para GND)
  • Verificar comunicação I2C com analisador lógico
  • Testar diode mode nos pontos de teste adjacentes

2. Remoção do Chip

  • Aplicar fluxo de solda de qualidade
  • Usar estação de ar quente a 350-380°C
  • Remover chip cuidadosamente sem danificar o PCB
  • Limpar resíduos de solda com wick e fluxo

3. Preparação do Novo Chip

  • Verificar se o chip é vanilla ou OTP
  • Se vanilla, programar EEPROM com dados corretos
  • Aplicar bolas de solda (reballing) se necessário
  • Tempo de reballing: 10-20 minutos com pasta de solda KC Micro Tronix [[13]]

4. Instalação

  • Alinhar cuidadosamente o chip no PCB
  • Aplicar calor uniforme até o assentamento
  • Deixar resfriar naturalmente
  • Verificar continuidade e curtos

5. Testes Pós-Reparo

  • Medir tensões LDO de saída
  • Testar comunicação I2C
  • Conectar carregador e verificar negociação PD
  • Testar carregamento da bateria
Atenção: Chips CD3215 podem ser reballed, mas requerem equipamento adequado e técnica precisa. Danos ao PCB são irreversíveis. [[13]]

📖 Dicionário de Kernel Panic

💥 O que é Kernel Panic?

Kernel Panic é uma medida de segurança do macOS quando o sistema operacional encontra um erro crítico do qual não pode se recuperar com segurança. É o equivalente macOS ao "Blue Screen of Death" do Windows. [[45]]

Thermal Emergency Superaquecimento Sensor térmico ou cooler Verificar sensores I2C, ventoinhas

🚨 iBoot Panic ANS2

🔍 ANS2 Panic

O panic ANS2 está relacionado ao AppleNVMeStorage, indicando problemas com o controlador SSD NVMe. É frequentemente causado por corrupção de dados ou falha de hardware no SSD. [[44]]

Causas Comuns
  • SSD NVMe defeituoso ou corrompido
  • Problema de comunicação I2C com controlador SSD
  • Firmware do SSD corrompido
  • Conflito de driver/kernel
  • Problema de energia no barramento NVMe
🔧 Procedimento de Diagnóstico
  1. Boot em Safe Mode: Segurar Shift durante boot
  2. Apple Diagnostics: Segurar D durante boot
  3. Target Disk Mode: Verificar se SSD é detectado
  4. DFU Restore: Restaurar via Apple Configurator 2
  5. Substituir SSD: Se persistir, possível falha de hardware
Importante: Em MacBooks com SSD soldado, a substituição requer equipamento especializado e pode não ser viável economicamente.

⏱️ Watchdog Timeout

🐕 O que é Watchdog?

O watchdog é um timer de hardware que monitora se o sistema está respondendo. Se o kernel ou um driver crítico parar de responder dentro de um período específico (tipicamente 2-5 minutos), o watchdog triggera um panic para prevenir corrupção de dados. [[51]]

📋 Tipos de Watchdog Panic
Tipo Timeout Causa
Kernel Watchdog 120 segundos Kernel thread travada
Scheduler Watchdog Variable CPU não responde
SMC Watchdog 30-60 segundos SMC não responde
AOP Watchdog Variable AOP processor hung
Dica: Para diagnosticar watchdog panics, habilite keepsyms=1 e debug=0x100 nas boot args. Isso preserva símbolos de kernel no panic report. [[105]]

🔄 AOP — Always On Processor

🧠 O que é AOP?

O Always-On Processor (AOP) é um coprocessador de baixo consumo presente em dispositivos Apple que permanece ativo mesmo quando o dispositivo está em sleep ou desligado. Ele gerencia tarefas como "Hey Siri", sensores de movimento, e monitoramento de eventos de wake. [[97]]

⚙️ Funções do AOP
  • Processamento de voz "Hey Siri" sempre ativo
  • Gerenciamento de sensores (accelerometer, gyro)
  • Monitoramento de eventos de wake (abertura de tampa, teclado)
  • Processamento de áudio/vídeo em background
  • Comunicação com periféricos durante sleep
  • Gerenciamento de energia durante idle states
🚨 AOP Panic

Panics relacionados ao AOP geralmente ocorrem durante estados de idle/sleep e podem estar ligados a problemas de gerenciamento de energia, áudio, ou sensores. [[105]]

SOLUÇÕES

  • Atualizar macOS para versão mais recente
  • Reset NVRAM/PRAM
  • Verificar kexts de terceiros
  • Restaurar macOS via Recovery
  • Em casos extremos, possível falha de hardware
Curiosidade: O AOP é o motivo pelo qual MacBooks M1/M2 podem "acordar" automaticamente ao abrir a tampa, mesmo desligados. Eles nunca estão completamente desligados. [[103]]

🔥 Thermal / Hot

🌡️ Monitoramento Térmico

O sistema de gerenciamento térmico em Macs utiliza múltiplos sensores I2C/SMBus para monitorar temperaturas de CPU, GPU, SSD, e outros componentes. Quando temperaturas críticas são atingidas, o sistema toma ações para prevenir danos. [[121]]

Componente Temp Normal Temp Alta Temp Crítica Ação
CPU 40-70°C 70-90°C >95°C Throttle/Shutdown
GPU 40-75°C 75-90°C >95°C Throttle/Shutdown
SSD NVMe 30-50°C 50-70°C >75°C Throttle performance
Bateria 20-35°C 35-45°C >50°C Reduzir carregamento
🔧 Diagnóstico de Problemas Térmicos
  • Verificar sensores de temperatura via I2C (endereços 0x48-0x4F)
  • Testar ventiladores e controle PWM
  • Limpar poeira de heatsinks e vents
  • Verificar pasta térmica da CPU/GPU
  • Testar sensores térmicos individuais
  • Verificar comunicação SMC-sensores
Thermal Emergency Panic: Se o sistema atingir temperatura crítica, um kernel panic será triggerado para prevenir dano permanente ao hardware.

🔐 SEP — Security

🛡️ Secure Enclave Processor

O Secure Enclave Processor (SEP) é um subsistema de segurança dedicado presente em iPhones, iPads, Macs modernos e outros dispositivos Apple. É isolado do processador principal para fornecer uma camada extra de segurança. [[108]]

🔑 Funções do SEP
  • Geração e armazenamento de chaves criptográficas
  • Processamento de Touch ID e Face ID
  • Proteção de dados biométricos
  • Secure boot e verificação de integridade
  • FileVault encryption keys
  • Keychain protection
  • Apple Pay secure element
Segurança: O SEP é isolado do processador principal e tem seu próprio microkernel seguro. Mesmo se o sistema principal for comprometido, os dados no SEP permanecem protegidos. [[114]]
🔍 SEP e I2C

O SEP se comunica com outros componentes via barramentos seguros, incluindo I2C/SMBus para: [[112]]

  • Comunicação com sensores biométricos
  • Interface com Secure Element
  • Monitoramento de integridade do sistema

⚙️ SMC — Power

⚡ Gerenciamento de Energia

O System Management Controller (SMC) é responsável por todas as funções de energia em Macs Intel, incluindo power-on sequence, gerenciamento de bateria, e distribuição de energia para componentes. [[88]]

Funções de Power do SMC
Função Descrição
Power On Sequence Controla sequência de inicialização de tensões
Battery Management Carregamento, descarga, e proteção
Power Rails Habilita/desabilita rails de alimentação
Sleep/Wake Gerencia transições de energia
AC Power Detection Detecta conexão de carregador
LED Status Controla LED de carga/status
🔧 Reset SMC

Resetar o SMC pode resolver problemas de energia, térmicos, e de bateria. [[92]]

  1. Desligar o Mac
  2. Segurar Shift + Control + Option (lado esquerdo) + Power por 10 segundos
  3. Soltar todas as teclas
  4. Ligar o Mac normalmente
Nota: Macs com Apple Silicon não têm SMC tradicional. As funções são integradas no SoC e não requerem reset manual. [[95]]

📡 I2C — Communication

📡 Protocolo de Comunicação

O I2C é um protocolo de comunicação serial síncrono que usa apenas duas linhas: SDA (dados) e SCL (clock). É amplamente utilizado para comunicação com sensores, EEPROMs, e controladores em sistemas embarcados. [[76]]

📋 Formato de Transmissão
Transmissão I2C Completa:

1. START Condition
   SDA: High → Low (enquanto SCL = High)

2. Slave Address (7 bits) + R/W bit (1 bit)
   - Bits 7-1: Endereço do slave
   - Bit 0: 0=Write, 1=Read

3. ACK/NACK (1 bit)
   - ACK: Slave puxa SDA low
   - NACK: SDA permanece high

4. Data (8 bits)
   - MSB primeiro

5. ACK/NACK

6. STOP Condition
   SDA: Low → High (enquanto SCL = High)

Exemplo - Write Byte:
[START] [0x42+W] [ACK] [0x05] [ACK] [0xFF] [ACK] [STOP]

Exemplo - Read Byte:
[START] [0x42+W] [ACK] [0x05] [ACK] 
[RESTART] [0x42+R] [ACK] [DATA] [NACK] [STOP]
🔍 Debugging I2C
  • Osciloscópio: Visualizar formas de onda SDA/SCL
  • Analisador Lógico: Decodificar pacotes I2C
  • Multímetro: Verificar pull-ups (deve haver ~3.3V)
  • i2c-tools (Linux): i2cdetect -y 1
  • IORegistryExplorer (macOS): Ver dispositivos I2C
Dica: Se não houver comunicação I2C, verifique primeiro os resistores de pull-up. Sem pull-up adequado, as linhas ficam flutuantes e a comunicação falha.

🔧 Soluções Comuns

  • Reset SMC: Conforme descrito na seção SMC
  • Safe Mode: Shift no boot para diagnóstico

  • Remove Kexts: Remover kexts de terceiros problemáticos
Nota: Muitos bugs de I2C em Hackintosh são causados por configurações incorretas de ACPI/SSDT. Verificar DSDT e patches necessários.
